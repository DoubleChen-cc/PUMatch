## PUMatch

GPU-accelerated subgraph matching prototype for large graphs (current entry point: `test.cu`, output executable: `test`).

### Requirements

- **OS**: Linux / WSL2 recommended (data preparation scripts are bash and require `wget/gzip`)
- **CUDA**: CUDA **>= 12.8** (see root `Makefile`), GPU architecture **>= sm_70**
- **Build tools**:
  - Root: `make` + `nvcc` (CUDA Toolkit)
  - `graph_converter/`: `make` + `g++` + `gcc` (uses `-fopenmp`)
- **Utilities**: `bash`, `wget`, `gzip`

### Quick start

#### 1) Prepare data (download + convert to binary graph)

From the project root:

```bash
cd graph_converter
make
bash prepare_data.sh
```

What `prepare_data.sh` does:

- Downloads SNAP Friendster into `data/txt_graph/`
- Decompresses and copies the text graph to `data/bin_graph/<graph_name>/snap.txt`
- Runs `convert.sh` for each graph directory to generate the binary files needed by the main program (e.g., `snap.txt.meta.txt`, `snap.txt.vertex.bin`, `snap.txt.edge.bin`, `snap.txt.label.bin`, etc.)

#### 2) Build the main program (back to project root)

```bash
cd ..
make
```

This produces `./test`.

#### 3) Run

`test` takes two arguments:

1. **Graph prefix**: a path prefix *without* extensions (the program will read `<prefix>.meta.txt/.vertex.bin/.edge.bin/.label.bin`)
2. **Pattern file path**: e.g. `pattern/1.g`

Example (Friendster graph + one pattern):

```bash
./test data/bin_graph/com-friendster.ungraph/snap.txt pattern/1.g
```

Output:

- Column 1: GPU kernel time (ms)
- Column 2: match count

### Input formats

#### Pattern (`.g`)

`pattern/*.g` uses a simple gSpan-like text format:

- `v <id> <label>` defines a vertex
- `e <src> <dst> <label>` defines an edge

Example `pattern/1.g`:

```text
v 0 1
v 1 1
...
e 1 0 1
e 0 1 1
...
```

> Note: `PAT_SIZE = 7` in `src/config.h`, so the current implementation targets 7-vertex patterns by default.

#### Graph (binary)

Given `<prefix>`, the main program reads:

- `<prefix>.meta.txt`
- `<prefix>.vertex.bin`
- `<prefix>.edge.bin`
- `<prefix>.label.bin` (required when `LABELED=true` in `src/config.h`)

These files are generated by the pipeline in `graph_converter/convert.sh`.

### FAQ

- **GPU arch mismatch**: root `Makefile` defaults to `-arch=sm_80`. If your GPU is different, change it to the right `sm_XX` (e.g., `sm_70`/`sm_75`/`sm_86`/`sm_89`).
- **Doesn't run on native Windows**: data preparation is bash-based and depends on `wget/gzip`. Use WSL2 Ubuntu for the steps above.
